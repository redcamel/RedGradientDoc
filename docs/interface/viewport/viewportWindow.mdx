# Viewport Window

<hr/>

```Viewport Window```는 실제 그라디언트를 화면상에 편집하기 위한 공간입니다.

개별 뷰포트는 수정 할 Container 선택하고 해당 컨테이너를 기반으로 편집을 수행합니다.

<hr/>

<TOCInline toc={toc}/>

<hr/>

## Viewport Window Menu
![ui_viewport_window_menu](./img/ui_viewport_window_menu.png)
- #### 해당 뷰포트가 편집해야할 container를 지정합니다.
  - **Before** Container : css상의 ```::before``` 공간을 의미합니다.

		![ui_viewport_container_main](./img/ui_viewport_container_main.png)
  - **Main** Container : css를 표현할 ```메인 Dom```

		![ui_viewport_container_before](./img/ui_viewport_container_before.png)
  - **After** Container : css상의 ```::after``` 공간을 의미합니다.

		![ui_viewport_container_after](./img/ui_viewport_container_after.png)

:::info
RedGradient는 ```하나의 Dom에 표현될 그라디언트```를 편집합니다.
메인 Dom을 ```Main Container```로 선언합니다.
:::


- #### 개별 Viewport Window의 view 상태를 설정합니다.
  - **Device** Visible : ```가상 디바이스``` 표현 유무를 설정합니다.

		![ui_viewport_device_off](./img/ui_viewport_device_off.png)
		![ui_viewport_device_on](./img/ui_viewport_device_on.png)
	- **Ruler** Visible :  ```가상 Ruler```의 표현 유무를 설정합니다.

		![ui_viewport_ruler_off](./img/ui_viewport_ruler_off.png)
		![ui_viewport_ruler_on](./img/ui_viewport_ruler_on.png)
	- Allow **Overflow** : 가상 디바이스 사이즈내로 ```overflow 허용 유무```를 설정합니다.

		![ui_viewport_overflow_hidden](./img/ui_viewport_overflow_hidden.png)
		![ui_viewport_overflow_none](./img/ui_viewport_overflow_none.png)
  - Container **Background Color** : 편집중인 ```Container의 배경색```을 설정합니다.

		![ui_viewport_color](./img/ui_viewport_color.png)

## Transform Mode Helper
- 해당 레이어의 **편집 타겟을 설정**합니다.
  - ```container```
  - ```gradient```
  - ```border```
  - ```none```

### Container Mode
- 컨테이너 사이즈를 ```마우스 + 키보드```를 통해 수정 할 수 있습니다.

:::info
단축키 : ```CTR + 1```
:::
